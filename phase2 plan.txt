Phase 2 Execution Plan: Storefront Routing & Retail UX
The Architecture
ResellerStorefront Entity: A custom table linking a VendorId to a UrlSlug, CustomDomain, and branding elements (colors, logos).
IStorefrontContext: A scoped service that inspects every HTTP request (looking at the URL path or subdomain) and caches the active Storefront in memory.
INopRouteProvider: Registers custom dynamic routes without breaking standard nopCommerce routing.
Dynamic CSS Injection: A Widget that injects CSS variables (like --primary-color) into the <head> of the page, completely reskinning the master theme based on the active Reseller.

Step 3: Implement IStorefrontContext (The Request Sniffer)
This is the brain of Phase 2. It inspects the current HTTP request, extracts the slug, and holds the ResellerStorefront data in memory for the duration of the request.
Services/IStorefrontContext.cs
code
C#
public interface IStorefrontContext
{
    Task<ResellerStorefront> GetCurrentStorefrontAsync();
}
Services/StorefrontContext.cs
Inject IHttpContextAccessor and IStaticCacheManager.
Logic:
Read _httpContextAccessor.HttpContext.Request.Path.
Does it start with /store/{slug}?
If yes, extract the {slug}.
Query the database for the ResellerStorefront matching that slug.
Cache the result using nopCommerce's IStaticCacheManager so subsequent requests for the same storefront don't hit the database.
Return the ResellerStorefront.
Step 4: Custom Routing Engine
We need to register our /store/{slug} URL structure with the ASP.NET Core endpoint routing system without breaking nopCommerce's native catch-all routes.
Infrastructure/StorefrontRouteProvider.cs
code
C#
using Microsoft.AspNetCore.Builder;
using Microsoft.AspNetCore.Routing;
using Nop.Web.Framework.Mvc.Routing;

namespace Nop.Plugin.Marketplace.Storefront.Infrastructure
{
    public class StorefrontRouteProvider : INopRouteProvider
    {
        public void RegisterRoutes(IEndpointRouteBuilder endpointRouteBuilder)
        {
            // Maps /store/shoeking to the StorefrontController -> Index Action
            endpointRouteBuilder.MapControllerRoute(
                name: "MarketplaceStorefront",
                pattern: "store/{slug}/{action=Index}/{id?}",
                defaults: new { controller = "Storefront", action = "Index" }
            );
        }

        // Must be higher than 0 so it registers before the generic native routes
        public int Priority => 100; 
    }
}
Step 5: Dynamic Branding Injection (CSS Variables)
Instead of cloning physical HTML/CSS themes for every Reseller (which kills performance), we will inject a dynamic <style> block into the HTML <head>.
1. Create a ViewComponent (Components/StorefrontBrandingViewComponent.cs)
Inject IStorefrontContext.
If GetCurrentStorefrontAsync() is null, return an empty view.
If it exists, pass the ResellerStorefront model to the View.
2. Create the View (Views/StorefrontBranding/Default.cshtml)
code
Html
@model Nop.Plugin.Marketplace.Storefront.Domains.ResellerStorefront

<style>
    /* Override standard nopCommerce theme variables globally! */
    :root {
        --primary-color: @Model.PrimaryColorHex !important;
        --button-background: @Model.PrimaryColorHex !important;
    }
    
    /* Inject custom Logo */
    .header-logo a {
        background-image: url('@Url.Content("~/images/uploaded/" + Model.LogoPictureId)'); /* Simplified for example */
    }
</style>
3. Register the Widget Zone in the Plugin Class
In MarketplaceStorefrontPlugin.cs, implement IWidgetPlugin.
Map the ViewComponent to PublicWidgetZones.HeadHtmlTag.
Result: Whenever a consumer visits /store/shoeking/..., the <head> of the page is automatically injected with ShoeKing's colors and logo, instantly reskinning the entire website!
Step 6: The Storefront Homepage Controller
When a user visits /store/shoeking, they should see the Reseller's specific homepage, banner, and cloned products.
Controllers/StorefrontController.cs
code
C#
using Microsoft.AspNetCore.Mvc;
using Nop.Web.Framework.Controllers;

namespace Nop.Plugin.Marketplace.Storefront.Controllers
{
    public class StorefrontController : BasePluginController
    {
        private readonly IStorefrontContext _storefrontContext;
        private readonly IProductService _productService; // Core service

        public StorefrontController(IStorefrontContext storefrontContext, IProductService productService)
        {
            _storefrontContext = storefrontContext;
            _productService = productService;
        }

        public async Task<IActionResult> Index(string slug)
        {
            var storefront = await _storefrontContext.GetCurrentStorefrontAsync();
            if (storefront == null) return NotFound();

            // Fetch products OWNED by this specific Reseller
            // Because of Phase 1 "Light Cloning", these products are native to nopCommerce!
            var products = await _productService.SearchProductsAsync(
                vendorId: storefront.VendorId,
                visibleIndividuallyOnly: true
            );

            // Pass products and storefront data to a custom View
            return View("~/Plugins/Marketplace.Storefront/Views/Storefront/Index.cshtml", products);
        }
    }
}
